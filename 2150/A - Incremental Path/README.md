<div class="header"><div class="title">A. Incremental Path</div><div class="time-limit"><div class="property-title">time limit per test</div>1 second</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><div class="epigraph"><div class="epigraph-text"><span class="tex-font-style-it"><a href="https://soundcloud.com/rymdkraft/rymdsylt">Rymdkraft - Rymdsylt</a></span></div><div class="epigraph-source">⠀</div></div><p><span class="tex-font-style-bf">Hacks are disabled in this problem.</span></p><p>There is a strip containing $$$10^9$$$ cells, numbered from $$$1$$$ to $$$10^9$$$. Each cell is either black or white. Initially, $$$m$$$ distinct cells $$$a_1, a_2, \ldots, a_m$$$ are black, and the others are white.</p><p>If a person is on some cell $$$x$$$, he might be given two types of commands:</p><ul> <li> $$$\texttt{A}$$$: jump to the next cell, i.e. cell $$$x + 1$$$; </li><li> $$$\texttt{B}$$$: jump to the next white cell, i.e. the minimum $$$y > x$$$ such that cell $$$y$$$ is white. </li></ul><p>There is a string $$$s$$$ of length $$$n$$$, consisting of $$$n$$$ commands. For each $$$i$$$ from $$$1$$$ to $$$n$$$ in order, a person </p><ul> <li> starts from cell $$$1$$$; </li><li> executes the first $$$i$$$ commands in the string; </li><li> colors the last visited cell black (if it was already black, it remains black). </li></ul><p>You wonder which cells are black at the end of the process. Print them in increasing order.</p></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \le t \le 10^4$$$). The description of the test cases follows. </p><p>The first line of each test case contains two integers $$$n$$$ and $$$m$$$ ($$$1 \leq n \leq 10^5$$$, $$$1 \leq m \leq 10^5$$$) — the number of commands and the number of black cells at the beginning.</p><p>The second line of each test case contains a string $$$s$$$ of length $$$n$$$, consisting of characters $$$\texttt{A}$$$ and $$$\texttt{B}$$$, representing the commands.</p><p>The third line of each test case contains $$$m$$$ integers $$$a_1, a_2, \ldots, a_m$$$ ($$$1 \leq a_1 < a_2 < \ldots < a_m \leq 10^9$$$) — the initial black cells.</p><p>It is guaranteed that the sum of $$$n$$$ over all test cases does not exceed $$$10^5$$$, and the sum of $$$m$$$ over all test cases does not exceed $$$10^5$$$.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, print two lines:</p><ul> <li> in the first line, print a single integer $$$k$$$: the number of black cells at the end; </li><li> in the second line, print the labels of the black cells, in increasing order. </li></ul></div><div class="sample-tests"><div class="section-title">Example</div><div
